Propuesta de alcance F4a

F4a entrega solo Forma A (noticia) + el bloque flexible. Razones:

  • Forma A se usa en Home y archive-post → cobertura inmediata
  • Forma B (evento) tiene datos específicos (fecha+hora+lugar+CTA circular) acoplados al CPT agenda → encaja mejor en F4c
  • Mantiene F4a pequeño (~50 KB plan, 1 commit por task) como F1-F3
A

Solo Forma A (noticia) — recomendado

Bloque + card_noticia + sources manual + post + concurso. Agenda source + Forma B en F4c.

B

Ambas formas en F4a

Bloque + card_noticia + card_evento + 4 sources (manual + post + agenda + concurso). F4c solo añade archive-agenda + single-agenda.

C

Solo manual + post

F4a mínimo. Concurso source en F5. Más granular pero igual de plans.
